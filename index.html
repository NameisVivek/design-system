Design System
documentation

A token-driven, component-first design language. Warm stone canvas, indigo brand actions, high-density layout built for power users.

Project objective

This design system exists to give every product team a single, shared visual language — one set of decisions made once, used everywhere. By centralising colour, typography, spacing, and component behaviour into versioned tokens and documented patterns, teams spend less time debating pixels and more time solving user problems. The goal is consistency without rigidity: a foundation strong enough that any screen built on top of it feels immediately familiar, and flexible enough that teams can still ship fast.

Process
01
Design handoff
Started from a Claude Design export — a bundled prototype containing the final approved app shell, chat transcripts, and all design assets. Extracted tokens, icon fonts, and component structure from the bundle.
02
Token audit
Mapped the Supply token system — --s-* CSS custom properties covering colour, typography, spacing, radius, elevation, and iconography — into a structured reference so every decision traces back to a named token.
03
Documentation build
Rebuilt the project from a single app shell screen into a full documentation site — Foundation, Components, and Patterns sections — with live demos, searchable icon grids, and token tables for every element.
04
Mobile adaptation
Designed a parallel mobile layout — bottom tab navigation, sheet drawer, stacked content — ensuring the documentation and the framework shell both work as native-feeling apps on small screens without any zooming or horizontal overflow.
